# Этап 1 — 2.5D-движок и фундамент (детальный план)

Детализация Этапа 1 из `docs/DEVELOPMENT_PLAN.md`. Это **keystone**-этап: 2.5D-глубина + риг экипировки героя + хуки мир-структуры, на которые опираются все последующие этапы. Технические факты сверены по `game.js` (см. `docs/AI_HANDOFF.md` §0 и аудит в `docs/DEVELOPMENT_PLAN.md` §0).

**Согласованные решения (2026-06-17):**
- **Графика:** 2.5D-глубина «топ-даун+» (фасады, тени, параллакс, наклон). Без WebGL/фреймворков. ASCII-карты и коллизии — источник истины, не меняем.
- **Ассет-стратегия:** гибрид. Я строю движок с code/SVG-fallback (игра всегда работает и проходит visual smoke) + даю спеку; ты генеришь ключевой пиксель-арт во внешнем AI-редакторе; итерируем по каждому ассету.
- **Стиль:** пиксель-арт (рефы: Stardew Valley, CrossCode), единый стиль/палитра.

## 1. Цель и рамки

Превратить текущий плоский топ-даун в **псевдо-3D с глубиной**, сохранив достижимость и совместимость сейвов. По итогу Этапа 1:

- Объекты имеют высоту и **отбрасывают тень** при едином направлении света; корректная z-сортировка по глубине.
- За игровым полем — **параллакс-фон** (небо/горизонт/дальний план).
- Готов **риг экипировки героя** (слоевая композиция) — мост к Этапу 5 («видимые вещи»).
- Тайлы/зелень — в новом пиксель-стиле с глубиной; первый «вертикальный» билборд (дерево) как эталон пайплайна.
- Спящие хуки **день/ночь** (множитель освещения) под Reduced Motion.

Вне рамок Этапа 1 (идёт позже): полный спрайт-арт NPC — Этап 2; экстерьеры всех зданий артом — частично Этап 1C (каркас) + Этап 2; хаб/связность — Этап 3.

## 2. Технический подход 2.5D

В рамках текущего грида (`LOGICAL_TILE=32`, экранный `TILE=48`, `RENDER_SCALE=1.5`, canvas `1280×768`). Камера и карты остаются топ-даун; глубина создаётся **экранными приёмами**, без смены системы координат:

- **Поле высоты.** У зданий/деревьев/пропов/персонажей вводим `height` (логические px над землёй). Земля рисуется как сейчас; «высокие» объекты — как вертикальный фасад/билборд, «стоящий» на своём footprint.
- **Единый свет + тени.** Константа направления света (напр. сверху-слева). Хелпер `drawCastShadow(footprintX, footprintY, w, height)` рисует мягкую эллиптическую/скошенную тень на землю до отрисовки объекта. Кэш градиентов теней.
- **Z-сортировка по глубине.** Текущая сортировка по `y` в `drawCharacterLayer` расширяется на общий список «высоких» объектов (персонажи + деревья + пропы), ключ сортировки — `y + footprintOffset`. Здания рисуются слоем ниже, но их верхушки могут перекрывать — учесть порядок.
- **Билборды.** Высокие объекты — вертикальные спрайты с pivot в низ-центре (footprint). Лёгкий **форшортенинг** (объект «встаёт») — чисто визуальный сдвиг вверх, коллизия остаётся по footprint на карте.
- **Параллакс-фон.** Отдельный слой за `drawWorld()`: небо-градиент + дальние силуэты, скроллит на коэффициент < 1 от камеры. Региональный backdrop. Fallback — код-градиент.
- **День/ночь (hook).** Множитель освещения по времени суток поверх `drawAtmosphereOverlay()`. По умолчанию спящий/медленный; под Reduced Motion — статичный. Полноценный цикл и расписания NPC — Этап 3.
- **Производительность.** Пре-композиция статичных слоёв (фон, тени статичных зданий) в офскрин-кэш; лимиты параллакса/частиц; цель 60 FPS. Reduced Motion отключает анимацию.

**Инвариант стиля:** любой новый арт обязан откатываться к текущему примитивному рендеру (как `drawProp`/`drawTile`), иначе visual smoke «чернеет».

## 3. Пиксель-арт пайплайн и масштаб (важно)

`RENDER_SCALE = 1.5` — **нецелочисленный**. Если рисовать пиксель-арт, заготовленный в логическом размере (тайл 32px), движок отмасштабирует его ×1.5 → дробное масштабирование размывает пиксели.

**Решение (1:1 на экране):** авторить пиксель-арт в **экранном** разрешении = логический размер × 1.5. Тогда `drawImage(img, x, y, logW, logH)` под `ctx.scale(1.5)` ложится на устройство пиксель-в-пиксель — чётко.

- Тайл: логические `32×32` → **авторить `48×48` px**.
- Ячейка героя: логические `32×48` → **авторить `48×72` px** на кадр.
- Дерево: footprint ~1 тайл, высота ~1.5 тайла → **авторить ~`64×96` px**.
- Здание: footprint `W×H` тайлов → ширина фасада `W×48` px, высота по силуэту.

**Технические правила рендера:** для пиксель-ассетов `ctx.imageSmoothingEnabled = false` (nearest-neighbor) на время их отрисовки; не смешивать с уже существующими гладкими SVG в одном вызове без сброса флага.

**Тест-первый (де-риск):** прежде чем массово генерить арт, ты делаешь **1–2 тест-ассета** (тайл травы + дерево) по спеке; я прогоняю их через движок и подтверждаю чёткость/выравнивание/палитру. Только потом — массовая генерация. Это экономит часы переделок.

## 4. Спецификация ассетов (пиксель-арт)

Общие правила для всех:

- **Формат:** PNG с прозрачностью (альфа). AI/исходник — рядом в `*-src`-папке (как `assets/characters/portraits-src/`), в деплой не идёт.
- **Pivot:** низ-центр (ноги/footprint) для билбордов; тайлы — без pivot, по сетке.
- **Свет:** одно направление (фиксируем в `VISUAL_STYLE_GUIDE.md`), **без запечённой падающей тени** (движок рисует тень сам). Собственное затенение объёма — можно.
- **Палитра:** общая мастер-палитра (дам сватчи) + региональные подпалитры; держать ограниченный набор тонов на материал.
- **Бесшовность:** тайлы — тайлящиеся края; вариации не «шумят» (детерминируются `hashNoise`).
- **Имена:** под ключи `*_ASSETS` в `game.js` (напр. `tile-grass.png`, `tree-oak.png`, `building-townhall.png`, `hero-base.png`).
- **Папки:** `assets/tiles/`, `assets/characters/`, `assets/buildings/` (сейчас пуста), `assets/props/region/`.

| Ассет | Размер (px) | Footprint | Что входит | Что НЕ входит |
|---|---|---|---|---|
| Тайл (grass/road/plaza/water/dock/wall) | 48×48 | 1 тайл | базовый тайл + 2–4 вариации/декали | тени соседей (рисует автотайлинг) |
| Дерево/куст | ~64×96 | 1 тайл (низ-центр) | крона+ствол, собственное затенение | падающая тень (движок) |
| Герой: база | кадры 48×72, лист 4 напр.×N | 1 тайл | тело без одежды/в базовой | сменные слои (отдельно) |
| Герой: слои экипировки | те же 48×72, та же сетка/pivot | — | outfit/hat/hair/back/held по слоям | базовое тело |
| Здание по `kind` | (W×48)×высота | W×H тайлов | фасад+вход+крыша | падающая тень (движок), вывеска-текст |
| Backdrop региона | широкая полоса (напр. 1920×~300) | — | небо/горизонт/дальние силуэты слоями | интерактив |

Точные размеры зданий по `kind` и листы героя финализируем в под-этапах 1C/1E.
